在 hexo g 后可以直接通过 hexo d 进行部署，而无需先 hexo s，该命令只是用于本地预览而已

只有在部署到 GitHub Pages 的个人主页 https://username.github.io 上时，../_config.yml 中的deploy 的参数 branch 才为 master；如果你的网站不是放在该主页，而是放在子目录中如 https://username.github.io/academic 中时，需要先新建一个 gh-pages 的分支，并将 deploy 的参数改为 branch: gh-pages，然后在 repo 的设置中把 GitHub Pages 的发布分支选为 gh-pages

此外，在子目录中的网站还需要在 URL 相关的参数中设置网站的 url 及 root 为
